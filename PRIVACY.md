# Privacy Notice (Template)

Date: March 6, 2026

This template should be reviewed by your attorney before production use.

## Summary

MediaSorter is designed to process media locally on the user's Windows device.

## Data We Process

- File paths and filenames selected by the user
- Image/video content required for categorization
- Optional face-scan data used for local clustering (if enabled by user)

## Data Storage

MediaSorter stores app data locally (for example category settings, correction data, decision logs, and optional people clustering metadata) in the user's app-data directory.

## Data Sharing

- By default, MediaSorter does not require cloud upload for local sorting.
- Optional AI provider/runtime behavior should be disclosed clearly if any network calls are enabled.

## User Control

Users can:

- choose whether to enable face-scan features
- choose input/output folders
- remove local app data files manually

## Contact

For privacy requests, provide a contact email in your public README/release notes.
